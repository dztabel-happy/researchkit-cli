# Voice Pass 方法论（正式中文报告）

本文件是 voice pass 阶段的执行标准。目标读者是执行改写的 Agent。

定位先说清楚：通用"去 AI 味"方法（加第一人称、注入个性、允许跑题）**不适用**于正式研究报告。报告的去 AI 味只有一条主线：**把每一句模板腔、装饰腔、空泛腔，改写为具体的事实、数据、边界和动作**。改写过程中绝不允许编造新的事实、数字或来源；所有 `[^ev_001]` 这类稳定 evidence footnote token 和 `{{fig:*}}` / `{{tbl:*}}` label 必须原样保留。

## 一、AI 味模式清单

逐章通读，对照下面四层模式逐一处理。CLI 的 `lint-style` 只能拦截其中机械可判定的部分；语义层面的判断由本清单约束。

### 1. 词汇层

| 模式 | 例子 | 处理 |
| --- | --- | --- |
| 口号 / 营销词 | 赋能、打造闭环、降本增效、护城河、深耕、新篇章 | 删除或改为具体动作（CLI 阻塞） |
| 高频 AI 痕迹词 | 值得注意的是、不难发现、众所周知、毋庸置疑、蓬勃发展、未来可期 | 删除，直接说事实（CLI 阻塞） |
| 模板腔短语 | 综上所述、总而言之、在一定程度上、具有重要意义、深入探讨 | 改为有内容的总结句或删除（CLI 警告） |
| 意义拔高词 | 标志着、彰显了、体现了、开启了、里程碑 | 改为事实本身："成立于 2019 年"，而不是"标志着行业进入新阶段" |
| 同义词循环 | 同一对象在一章内轮换叫"该企业 / 这家公司 / 该厂商" | 全文统一为一个称谓，术语不换词 |

### 2. 句式层

| 模式 | 例子 | 处理 |
| --- | --- | --- |
| 机械串联 | 首先……其次……最后…… | 改用小标题或按逻辑关系（因果、对比、递进）组织（CLI 阻塞） |
| 否定式排比 | 这不仅是 X，更是 Y | 最多保留一处，其余直接陈述（CLI 按密度警告） |
| 三段式强凑 | "创新、效率与协同"式的凑三项 | 有几项写几项，两项或四项都可以 |
| 虚假范围 | "从芯片到生态"但两端不在同一量纲 | 改为具体列举 |
| 句尾挂空泛分词 | "……，反映了行业的深刻变革" | 删掉句尾拔高，或写出具体变了什么 |
| 过度限定堆叠 | "可能在某种程度上或将对……产生一定影响" | 一个判断只留一层限定："若 X 成立，则 Y 大概率上升" |

### 3. 结构层

| 模式 | 例子 | 处理 |
| --- | --- | --- |
| 粗体小标题列表 | `- **趋势一**：……` 连续堆叠 | 改为短段落、三级标题或表格（CLI 阻塞，≥4 个） |
| 破折号滥用 | 一段里多个"——"制造文案感 | 改逗号、冒号或拆句（CLI 按密度阻塞） |
| emoji / 装饰符号 | 🚀 ✅ | 全部删除（CLI 阻塞） |
| 连续 bullet 轰炸 | 十几条并列 bullet 代替论证 | 改为分析段落，bullet 只留可并列枚举的事实（CLI 阻塞，>8 连续） |
| 模板化"挑战与展望"结尾 | "尽管面临诸多挑战，前景依然广阔" | 写具体的风险、触发条件和应对动作 |
| 通用积极结论 | "未来可期，值得持续关注" | 写有边界的判断："若 Q4 渠道库存降至 1.5 个月以下，价格战趋缓" |

### 4. 内容层

| 模式 | 例子 | 处理 |
| --- | --- | --- |
| 模糊归因 | "研究表明 / 专家认为 / 数据显示"却无出处 | 写明具体来源并加 `[^n]`，追溯不到就删句或标注为假设（CLI 警告） |
| 套话开头 | "随着人工智能的快速发展……" | 直接从事实或本段判断切入（CLI 警告） |
| 无证据结论 | 判断句没有对应 evidence | 回到 gap 循环补证据，或明确标注为假设及其风险 |
| 空泛趋势总结 | "行业集中度不断提升" | 给出数字、时间窗和口径："CR5 从 2023 年的 41% 升至 2025 年的 55%[^3]" |

## 二、改写原则（报告版，按优先级）

1. **结论先行**：每段第一句是判断，不是铺垫。
2. **判断必须有边界**：写清成立条件、时间窗、置信度。"建议加强渠道管理"改为"建议在 Q4 前将渠道库存压降至 1.5 个月以内，否则按第 5 章测算毛利率将再降 2 个百分点"。
3. **具体优先于修饰**：删掉一个形容词，就补一个数字、日期或对象。
4. **信任读者**：删除"我们可以看到""接下来将分析"这类导览句，直接给内容。
5. **节奏变化**：连续三句等长就拆一句；长论证句和短判断句交替。
6. **删金句**：任何"值得被引用"感觉的句子都是装饰，删掉或改为平实陈述。
7. **术语一致**：同一概念全文只用一个名字；宁可重复，不可换词。

## 三、前后对照示例

**改写前**（AI 味）：

> 值得注意的是，随着新能源汽车行业的蓬勃发展，动力电池市场迎来了前所未有的机遇。这不仅体现在装机量的显著提升上，更标志着整个产业链的深度变革。展望未来，行业前景广阔，未来可期。

**改写后**（报告腔）：

> 2025 年国内动力电池装机量 486GWh，同比增长 32%，增速较 2024 年回落 9 个百分点。[^ev_004] 增长的主要约束已从产能转为碳酸锂价格波动：若价格维持在 9 万元/吨以下，二线厂商毛利率难以回到 10% 以上（测算见表 {{tbl:margin-model}}）。判断的边界是储能需求：若欧洲大储项目招标量低于预期，全年增速可能降至 20% 以下。

对照要点：删掉全部痕迹词；每个判断有数字和口径；"前景"改成了带触发条件的边界判断；引用和 label 落到了具体资产上。

## 四、执行流程

1. 按 section 逐章执行，不要一次性改全文（防止上下文过长导致风格漂移）。
2. 每章按"词汇层 → 句式层 → 结构层 → 内容层"过一遍清单。
3. 改写中发现无证据结论：能补证据的回 gap 循环，不能补的降级为显式假设。
4. 全部章节完成后运行 `research-kit lint-style`：机械错误（blocking）必须清零；warning 逐条判断，保留的要能向用户说明理由。
5. 完成逐章改写后重新合并，再登记当前 draft：`research-kit build-md --merge-sections`，随后运行 `research-kit build-md --voice-pass --voice-notes "<本次 voice pass 处理了哪些模式>"`。
6. voice pass 不改变事实与引用。如果改写后某句话失去了证据支撑，说明原句本来就不成立——这是 gap，不是文字问题。

## 五、快速自检清单

- 每章第一段第一句是判断吗？
- 全文还搜得到"值得注意/综上所述/首先其次最后/随着……的发展"吗？
- 每个"研究表明/数据显示"后面有 `[^n]` 吗？
- 有没有一段连续三句等长？
- 结尾一章是具体的风险 + 触发条件 + 动作，还是"前景广阔"？
- 删掉任何一个形容词，句子的信息量会变少吗？不会就删。
